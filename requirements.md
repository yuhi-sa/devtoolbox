## 1. プロジェクト概要

### 1.1 サイト名
**DevToolBox** (開発者ツールボックス)

### 1.2 コンセプト
開発者の日常作業を効率化する、シンプルで高速なWebツール集。

### 1.3 ターゲットユーザー
- 日本語圏のWeb開発者・プログラマー
- API開発、フロントエンド開発、ログ解析などを行うエンジニア
- コーディング中にちょっとした変換・検証を行いたいユーザー

### 1.4 プロジェクト目標
- GitHub Pages上で静的サイトとして公開
- 1日10人以上の安定的なオーガニック流入を獲得
- 将来的な広告収益化の基盤を構築

### 1.5 ホスティング制約
- GitHub Pages（静的ファイルのみ: HTML, CSS, JavaScript）
- サーバーサイド処理不可
- リポジトリ推奨上限 1GB、サイトサイズ最大 1GB
- 帯域ソフトリミット 100GB/月

## 2. 初期実装ツール

実装コスト・SEO効果・開発者ニーズのバランスを考慮し、以下の5ツールを初期リリース対象とする。

### 2.1 選定ツール一覧

| # | ツール名 | 選定理由 | 主要キーワード |
|---|---------|---------|--------------|
| 1 | JSON整形・検証 | 開発者の日常的ニーズ。検索ボリュームが安定して高い | 「JSON 整形 オンライン 無料」 |
| 2 | 文字数カウンター | 汎用性が高く開発者以外にもリーチ可能。実装が軽量 | 「文字数 カウント スペース含む」 |
| 3 | Base64エンコード/デコード | API開発で頻繁に使用。開発者ニーズが明確 | 「Base64 エンコード デコード オンライン」 |
| 4 | URLエンコード/デコード | Web開発の必須ツール。Base64と実装パターンが類似 | 「URLエンコード デコード オンライン」 |
| 5 | Unixタイムスタンプ変換 | ログ解析等で需要あり。ニッチだが競合が少ない | 「Unix タイムスタンプ 変換 ツール」 |

### 2.2 将来追加候補（優先度順）

| 優先度 | ツール名 | 備考 |
|--------|---------|------|
| 高 | ハッシュ生成（MD5, SHA-256） | Web Crypto APIで実装可能 |
| 高 | 正規表現テスター | リアルタイムマッチング表示 |
| 中 | QRコード生成 | 外部ライブラリ（qrcode.js等）が必要 |
| 中 | 色コード変換（HEX/RGB/HSL） | フロントエンド開発者向け |
| 低 | diff（テキスト比較） | 実装コストがやや高い |

### 3.2 各ページの要件

#### トップページ（index.html）
- サイトのキャッチコピーと概要説明
- ツール一覧をカード形式で表示（ツール名・概要・アイコン）
- 各ツールへのリンク
- 検索・フィルタ機能（ツール数が増えた際に備える、初期はシンプルな一覧）

#### ツールページ共通要件
- ツール本体（入力エリア + 出力エリア + 操作ボタン）
- ツールの使い方・説明テキスト（SEO対策としてテキストコンテンツを充実させる）
- 関連ツールへのリンク（内部リンク構造の強化）
- パンくずリスト
- コピーボタン（出力結果をクリップボードにコピー）
- クリアボタン（入力・出力をリセット）

#### About ページ
- サイトの目的・運営方針
- 全処理がクライアントサイドで完結する旨の説明（プライバシー面の安心感）

#### プライバシーポリシー
- データ収集に関する方針（クライアントサイド完結のため個人データは収集しない）
- 広告ネットワークによるCookie使用の説明
- Google Analytics等の分析ツール使用に関する記載

#### 利用規約
- 免責事項（ツールの出力結果に関する責任の所在）
- 利用条件

## 4. 各ツール機能仕様

### 4.1 JSON整形・検証ツール

**機能要件:**
- JSON文字列の入力（テキストエリア）
- 整形（Pretty Print）: インデントスペース数を選択可能（2/4スペース、タブ）
- 圧縮（Minify）: 改行・空白を除去
- 検証（Validate）: 構文エラーの検出とエラー位置の表示
- 入力エリアへのペースト時に自動整形するオプション

**非機能要件:**
- 大きなJSONファイル（~1MB程度）でもブラウザがフリーズしないこと
- エラーメッセージは日本語で表示

### 4.2 文字数カウンター

**機能要件:**
- テキスト入力（テキストエリア）
- リアルタイムカウント表示:
  - 文字数（スペース含む / スペース除外）
  - 行数
  - 単語数
  - バイト数（UTF-8）
- 入力と同時にリアルタイム更新

**非機能要件:**
- 大量テキスト（10万文字以上）でも遅延なくカウント

### 4.3 Base64エンコード/デコード

**機能要件:**
- テキスト入力 → Base64エンコード出力
- Base64入力 → テキストデコード出力
- エンコード/デコードの切り替えボタンまたはタブ
- 文字コード選択（UTF-8をデフォルト）
- 不正なBase64文字列入力時のエラー表示

**非機能要件:**
- 日本語（マルチバイト文字）の正確なエンコード/デコード

### 4.4 URLエンコード/デコード

**機能要件:**
- テキスト入力 → URLエンコード出力
- URLエンコード入力 → テキストデコード出力
- エンコード/デコードの切り替え
- encodeURIComponent / encodeURI の両方に対応（選択可能）
- 不正な入力時のエラー表示

**非機能要件:**
- 日本語URLの正確な変換

### 4.5 Unixタイムスタンプ変換

**機能要件:**
- Unixタイムスタンプ（秒/ミリ秒） → 人間が読める日時形式
- 日時入力 → Unixタイムスタンプ変換
- 現在時刻のタイムスタンプをワンクリックで取得
- タイムゾーン選択（デフォルト: ローカルタイムゾーン、UTC切り替え可能）
- 複数の日時フォーマットでの出力（ISO 8601、日本標準形式等）

**非機能要件:**
- 秒とミリ秒の自動判別

## 5. 技術仕様

### 5.1 基本技術スタック

| 項目 | 技術 |
|------|------|
| マークアップ | HTML5 |
| スタイル | CSS3（CSS Custom Properties使用） |
| スクリプト | Vanilla JavaScript（ES6+） |
| フレームワーク | 不使用（軽量化のため） |
| ビルドツール | 不使用（ファイルをそのまま配信） |
| ホスティング | GitHub Pages |

### 5.2 ブラウザ対応

| ブラウザ | バージョン |
|---------|-----------|
| Chrome | 最新2バージョン |
| Firefox | 最新2バージョン |
| Safari | 最新2バージョン |
| Edge | 最新2バージョン |
| モバイルブラウザ | iOS Safari、Android Chrome 最新版 |

### 5.3 外部ライブラリ方針
- 原則として外部ライブラリは使用しない
- 将来的にQRコード生成など、Vanilla JSでの実装が非現実的な機能に限りCDN経由で軽量ライブラリを許可
- 使用する場合はライブラリサイズとライセンスを確認すること

### 5.4 パフォーマンス目標

| 指標 | 目標値 |
|------|--------|
| Lighthouse Performance スコア | 95以上 |
| First Contentful Paint (FCP) | 1.0秒以下 |
| Largest Contentful Paint (LCP) | 1.5秒以下 |
| Total Blocking Time (TBT) | 50ms以下 |
| Cumulative Layout Shift (CLS) | 0.05以下 |
| ページサイズ（HTML+CSS+JS合計） | 各ページ100KB以下（非圧縮） |

### 5.5 処理方針
- 全ての処理はクライアントサイド（ブラウザ内）で完結する
- サーバーへのデータ送信は一切行わない
- ユーザーが入力したデータはブラウザのメモリ上でのみ処理し、永続化しない

## 7. 広告配置計画

### 7.1 方針
- ツール提供が主目的であり、広告は副次的な要素として控えめに配置
- 広告はプレースホルダー（div要素）として実装し、広告スクリプトは後から差し替え可能にする
- ユーザー体験を損なわない配置を優先

### 7.2 広告枠配置

#### ツールページ

```
┌─────────────────────────────────────────────┐
│  ヘッダー（ナビゲーション）                      │
├─────────────────────────────────────────────┤
│  [広告枠 A: バナー 728x90 / モバイル 320x100]   │
├──────────────────────────┬──────────────────┤
│                          │                  │
│  ツール本体              │ [広告枠 C]        │
│  （入力・出力エリア）     │  サイドバー広告   │
│                          │  300x250         │
│                          │  (デスクトップ    │
│                          │   のみ表示)       │
├──────────────────────────┤                  │
│                          │                  │
│  ツール説明テキスト       │                  │
│  （使い方・関連情報）     │                  │
│                          │                  │
├──────────────────────────┴──────────────────┤
│  [広告枠 B: バナー 728x90 / モバイル 320x100]   │
├─────────────────────────────────────────────┤
│  フッター                                     │
└─────────────────────────────────────────────┘
```

#### トップページ
- ヘッダー下バナー（広告枠A）のみ配置
- ツール一覧の視認性を優先

### 7.3 広告実装方式

```html
<!-- 広告プレースホルダーの実装例 -->
<div class="ad-slot" id="ad-header" data-ad-size="728x90">
  <!-- 広告スクリプトをここに挿入 -->
</div>
```

- 各広告枠はclass="ad-slot"のdiv要素として配置
- data属性で推奨サイズを指定
- 広告未設定時は非表示（高さ0）とし、レイアウトシフトを防ぐ

### 7.4 広告ネットワーク導入計画

| フェーズ | 時期 | 広告ネットワーク | 条件 |
|---------|------|----------------|------|
| Phase 1 | 初期 | なし（プレースホルダーのみ） | サイト公開・SEO基盤整備に集中 |
| Phase 2 | 月3-4 | Adsterra / PropellerAds | 審査なしで即開始可能 |
| Phase 3 | 月7以降 | Google AdSense | カスタムドメイン取得後、十分なコンテンツ蓄積後に申請 |

## 8. SEO要件

### 8.1 メタデータ

各ページに以下を固有に設定する:

| 要素 | 仕様 |
|------|------|
| title | 60文字以内。形式: 「{ツール名} - DevToolBox」 |
| meta description | 120文字以内。ツールの機能を簡潔に説明 |
| canonical URL | 各ページの正規URLを指定 |
| lang属性 | `lang="ja"` |

**タイトル例:**
- トップ: 「DevToolBox - 開発者向け無料オンラインツール集」
- JSON整形: 「JSON整形・検証ツール（オンライン無料） - DevToolBox」
- 文字数カウンター: 「文字数カウンター（スペース含む/除く対応） - DevToolBox」

### 8.2 OGPタグ

各ページに以下のOpen Graphタグを設定:

```html
<meta property="og:title" content="{ページタイトル}">
<meta property="og:description" content="{ページ説明}">
<meta property="og:type" content="website">
<meta property="og:url" content="{ページURL}">
<meta property="og:site_name" content="DevToolBox">
<meta property="og:locale" content="ja_JP">
```

### 8.3 構造化データ（JSON-LD）

各ツールページに`WebApplication`スキーマを設定:

```json
{
  "@context": "https://schema.org",
  "@type": "WebApplication",
  "name": "JSON整形・検証ツール",
  "description": "JSONデータを見やすく整形・検証するオンラインツール",
  "url": "{ツールURL}",
  "applicationCategory": "DeveloperApplication",
  "operatingSystem": "All",
  "offers": {
    "@type": "Offer",
    "price": "0",
    "priceCurrency": "JPY"
  },
  "isPartOf": {
    "@type": "WebSite",
    "name": "DevToolBox",
    "url": "{サイトURL}"
  }
}
```

### 8.4 sitemap.xml

- 全ページのURLを記載
- lastmod、changefreq、priorityを適切に設定
- ツール追加時に手動で更新

### 8.5 robots.txt

```
User-agent: *
Allow: /
Sitemap: {サイトURL}/sitemap.xml
```

### 8.6 内部リンク戦略
- 各ツールページに「関連ツール」セクションを設置
- パンくずリスト: トップ > {ツール名}
- フッターに全ツールへのリンク一覧

### 8.7 コンテンツSEO
- 各ツールページに以下のテキストコンテンツを併設:
  - ツールの使い方（300文字以上）
  - 関連する技術知識の解説（500文字以上）
  - よくある質問（FAQ）（2-3項目）
- 将来的なAdSense審査を見据え、各ページの総テキスト量は800文字以上を目標

## 10. 非機能要件

### 10.1 アクセシビリティ
- セマンティックHTML（header, main, nav, footer, section等）を適切に使用
- フォームラベルの適切な設定
- キーボード操作でツールが使用可能
- 十分なカラーコントラスト比（WCAG 2.1 AA基準）

### 10.2 セキュリティ
- ユーザー入力データをサーバーに送信しない
- XSS対策: ユーザー入力をDOMに挿入する際は適切にエスケープ
- 外部スクリプトの読み込みは広告関連のみに限定

### 10.3 プライバシー
- ユーザーが入力したデータは一切サーバーに送信・保存しない
- 全ての変換・処理はブラウザ上で完結
- この方針をAboutページおよびプライバシーポリシーで明示

## 12. ファイル命名・コーディング規約

### 12.1 ファイル命名
- HTMLファイル: ケバブケース（例: `json-formatter/index.html`）
- CSSファイル: ケバブケース（例: `style.css`）
- JSファイル: ケバブケース（例: `json-formatter.js`）

### 12.2 CSS
- CSS Custom Properties（変数）でカラー・スペーシングを管理
- BEM命名規則に準拠（例: `.tool-card__title--active`）
- メディアクエリはモバイルファーストで記述

### 12.3 JavaScript
- `"use strict"` を使用
- グローバルスコープの汚染を避ける（IIFE or モジュールパターン）
- DOM操作は `DOMContentLoaded` イベント後に実行
- エラーハンドリングはtry-catchで適切に処理
